SCADA System Pentest Report (Anonymized)

Overview

A white-box security assessment was conducted on an industrial SCADA (Supervisory Control and Data Acquisition) environment. The system included PLC-controlled field devices and a web-based HMI for remote supervision. The testing revealed multiple real-world vulnerabilities with potentially critical impact on operational integrity.

1. Cross-Site Scripting (XSS)

Description: The SCADA HMI interface accepted unescaped input, making it vulnerable to both stored and reflected XSS.

Impact:

  • Execution of arbitrary JavaScript
  • Session hijacking
  • Token theft and UI manipulation

Recommendation: Implement strict input validation, sanitize outputs before rendering, and use Content Security Policy (CSP).

2. Privilege Escalation

Description: Access control flaws allowed lateral and vertical privilege escalation, permitting unauthorized users to access sensitive functions.

Impact:

  • Unauthorized access to admin functionality
  • System configuration changes
  • Process manipulation

Recommendation: Enforce strict role-based access controls and segregate admin features from user context.

3. Insecure JWT Configuration

Description: JWTs issued post-authentication had excessive expiration times, increasing the risk of token misuse.

Impact:

  • Persistent access via stolen tokens
  • Difficult token revocation

Recommendation: Reduce token lifespan, use refresh tokens, and implement rotation & revocation mechanisms.

Engagement Details

  • Mode: White-Box Testing
  • Scope: Web-based SCADA HMI + auth components
  • Environment: Industrial control with PLC integration
