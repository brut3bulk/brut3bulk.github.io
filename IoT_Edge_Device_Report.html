IoT/Edge Device Security Report (Anonymized)

Summary

A grey-box security assessment was conducted on an industrial IoT edge device used to collect and transmit sensor data. The device included network services, embedded credentials, and API endpoints. Multiple real-world vulnerabilities were identified.

1. ARP Spoofing / Network Poisoning

Description: The device accepted ARP replies without verification, enabling ARP poisoning and MitM attacks.

Impact:

  • Session hijacking
  • Interception of credentials and control traffic

Recommendation: Harden network stack, use ARP inspection and encrypted channels.

2. DDoS Vulnerability Across Network Layers

Description: The device was susceptible to multi-layered DDoS attacks:

  • Layer 3: Volumetric flooding exhausted network interfaces
  • Layer 4: SYN floods consumed socket states
  • Layer 7: GET/POST floods impacted API performance
  • TLS Exhaustion: Repeated SSL handshakes drained CPU resources

Recommendation: Implement rate-limiting, SYN cookies, TLS hardening, and request validation.

3. Weak Password Hashing (MD5)

Description: User credentials were hashed using MD5, an insecure and outdated algorithm.

Impact:

  • Susceptibility to brute-force and collision attacks
  • Credential recovery and account takeover

Recommendation: Use bcrypt or SHA-256 with salt and pepper. Enforce secure password policies.

Engagement Details

  • Mode: Grey-Box Testing
  • Scope: Industrial IoT edge gateway (live deployment)
  • Environment: Connected sensors, real-time data processing

Note: This report is anonymized. All findings reflect real vulnerabilities identified in a secured ethical context. No device, vendor, or client identities are disclosed.
